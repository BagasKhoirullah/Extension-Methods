📌 Review Sesi 8 – Builder Pattern

  • Builder Pattern untuk membangun objek kompleks secara bertahap.
  • Method chaining untuk menciptakan fluent API yang mudah dibaca.
  • Contoh: Pizza.builder().size('large').addTopping('cheese').build()

Contoh cepat:

class ConfigBuilder {
    ConfigBuilder host(String h) { return this; }
    ConfigBuilder port(int p) { return this; }
    Config build() { return Config(); }
}

🧠 Pengantar Extension Methods

  • Apa itu? Fitur untuk menambahkan metode baru ke tipe yang sudah ada tanpa memodifikasi kode asli.
  • Tersedia sejak Dart 2.7.
  • Analog: Memberikan kemampuan baru tanpa mengubah benda aslinya.

💡 “Extension methods memungkinkan kita memperluas perilaku tipe yang sudah ada dengan cara yang natural dan type-safe.”

⚠️ Masalah Tanpa Extension Methods

Tanpa extension, kita sering membuat utility class yang kurang natural dan terasa asing saat digunakan.

// Utility class yang kurang natural
class StringUtils {
    static String capitalize(String text) {
        if (text.isEmpty) return text;
        return text[0].toUpperCase() + text.substring(1);
    }
    static bool isEmail(String text) {
        return text.contains('@');
    }
}

void main() {
    // Penggunaan kurang natural
    var name = 'john';
    var capitalized = StringUtils.capitalize(name); // 'John'
    var isEmail = StringUtils.isEmail('test@mail.com'); // true

    print('Original name: $name');
    print('Capitalized name: $capitalized');
    print('Is "test@mail.com" an email? $isEmail');
}

🔴 Kode terasa imperatif dan tidak mengikuti gaya object-oriented yang natural.

✅ Solusi dengan Extension Methods

Dengan extension, kita bisa menambahkan method langsung ke tipe String sehingga penggunaan menjadi lebih natural dan fluent.

// Extension method untuk String
extension StringExtensions on String {
    String capitalize() {
        if (isEmpty) return this;
        return this[0].toUpperCase() + substring(1);
    }
    bool get isEmail => contains('@');
    String reverse() {
        return split('').reversed.join();
    }
}

void main() {
    // Penggunaan lebih natural
    var name = 'john';
    var capitalized = name.capitalize(); // 'John'
    var isEmail = 'test@mail.com'.isEmail; // true
    var reversed = 'hello'.reverse(); // 'olleh'

    print('Original name: $name');
    print('Capitalized name: $capitalized');
    print('Is "test@mail.com" an email? $isEmail');
    print('Reversed "hello": $reversed');
}

✅ Kode lebih readable, expressive, dan mengikuti paradigma OOP.

📐 Sintaks Extension Methods

Bentuk dasar dari extension methods sangat sederhana:

extension NamaExtension on TipeYangDiperluas {
    // Method, getter, setter, atau operator tambahan
    ReturnType namaMethod(Parameter) {
        // ...
    }
}
  • NamaExtension (opsional) – jika dihilangkan, extension hanya tersedia di library yang sama.
  • TipeYangDiperluas – tipe yang ingin kita tambahi fungsionalitas (misal String, int, atau kelas kustom).
  • Di dalam body, kita bisa mengakses this yang merujuk pada instance dari tipe tersebut.

Contoh sederhana:

extension IntExtensions on int {
    bool get isEven => this % 2 == 0;
    bool get isOdd => !isEven;
}

void main() {
    print(4.isEven); // true
    print(7.isOdd);  // true
}

📌 Extension juga bisa memiliki getter, setter, dan operator overloading.

📦 Tipe yang Didukung

Extension methods dapat diterapkan pada berbagai macam tipe di Dart:

  • Built-in types: String, int, double, List, Map, dll.
  • Custom classes: Kelas buatan sendiri.
  • Generic types: List<T>, Map<K, V>.
  • Nullable types: String?, int?.

Contoh: Extension untuk List<int>

extension IntListExtensions on List {
  int get sum => fold(0, (total, item) => total + item);
  double get average => isEmpty ? 0 : sum / length;
}

void main() {
  var numbers = [1, 2, 3, 4, 5];
  print(numbers.sum);      // 15
  print(numbers.average);  // 3.0
}

🏷️ Extension untuk Custom Classes

Kita juga bisa menambahkan extension ke kelas buatan sendiri, misalnya untuk mempermudah operasi atau menambah utilitas.

class User {
    final String firstName;
    final String lastName;
    final int age;

    User(this.firstName, this.lastName, this.age);
}

extension UserExtensions on User {
    String get fullName => '$firstName $lastName';
    bool get isAdult => age >= 18;
    String introduce() => 'Hi, I am $fullName, $age years old.';
}

void main() {
    var user = User('John', 'Doe', 25);
    print(user.fullName);   // John Doe
    print(user.isAdult);    // true
    print(user.introduce()); // Hi, I am John Doe, 25 years old.
}

🧩 Dengan extension, kode klien menjadi lebih ekspresif dan terorganisir tanpa harus mengubah kelas asli.

🛡️ Extension untuk Null Safety

Extension juga bisa diterapkan pada nullable types untuk memberikan akses yang aman.

extension NullableStringExtensions on String? {
    // Safe access untuk nullable string
    String orEmpty() => this ?? '';
    bool get isNullOrEmpty => this == null || this!.isEmpty;
    bool get isNotNullOrEmpty => !isNullOrEmpty;
    String withDefault(String defaultValue) => this ?? defaultValue;
}

void main() {
    String? name;
    print(name.isNullOrEmpty);      // true
    print(name.orEmpty());          // ''
    print(name.withDefault('Guest')); // 'Guest'
}

🛡️ Menghindari null checks yang berulang-ulang dan membuat kode lebih aman.

💰 Praktik 1 – Utilities untuk Dart Types

Membuat extension untuk num dengan berbagai utility yang berguna.

extension NumberExtensions on num {
  // Format ke Rupiah
  String toRupiah() {
    return 'Rp${toStringAsFixed(0).replaceAllMapped(
      RegExp(r'(\d{1,3})(?=(\d{3})+(?!\d))'),
      (match) => '${match[1]}.',
    )}';
  }

  // Check jika dalam range
  bool isBetween(num min, num max) {
    return this >= min && this <= max;
  }

  // Pangkat (untuk eksponen integer non-negatif)
  num power(int exponent) {
    if (exponent == 0) return 1;
    if (exponent < 0) throw ArgumentError('Exponent must be non-negative');
    var result = this;
    for (var i = 1; i < exponent; i++) {
      result *= this;
    }
    return result;
  }
}

void main() {
  var price = 1500000;
  print(price.toRupiah());      // 'Rp1.500.000'
  print(5.isBetween(1, 10));    // true
  print(2.power(3));            // 8
  print(2.power(0));            // 1 (tambahan contoh)
}

🧬 Generic Extensions

Extension juga bisa bersifat generic sehingga bekerja untuk berbagai tipe elemen.

extension ListExtensions on List {
  // Safe get dengan default (mengembalikan null jika index tidak valid)
  T? getOrNull(int index) {
    if (index < 0 || index >= length) return null;
    return this[index];
  }

  // Membagi list menjadi potongan (chunks) berukuran tertentu
  List> chunk(int size) {
    var chunks = >[];
    for (var i = 0; i < length; i += size) {
      chunks.add(sublist(i, i + size > length ? length : i + size));
    }
    return chunks;
  }

  // Mengembalikan elemen unik (menghilangkan duplikat)
  List distinct() {
    var result = [];
    for (var item in this) {
      if (!result.contains(item)) {
        result.add(item);
      }
    }
    return result;
  }
}

void main() {
  var numbers = [1, 2, 3, 4, 5, 6, 7];
  print(numbers.chunk(3));  // [[1, 2, 3], [4, 5, 6], [7]]
  print([1, 2, 2, 3, 3, 3].distinct()); // [1, 2, 3]
}

🔒 Extension Tanpa Nama

Extension tanpa nama hanya bisa diakses di library yang sama (satu file).

// File: date_utils.dart
extension on DateTime {
    String get toYYYYMMDD {
        return '${year}-${month.toString().padLeft(2, '0')}-${day.toString().padLeft(2, '0')}';
    }
    DateTime get startOfDay {
        return DateTime(year, month, day);
    }
}

void main() {
    // Hanya bisa diakses di file ini
    var today = DateTime.now();
    print(today.toYYYYMMDD);   // '2024-01-22'
    print(today.startOfDay);   // 2024-01-22 00:00:00.000
}

🔒 Cocok untuk utility internal yang tidak perlu diekspos ke publik.

⚡ Konflik Nama Extension

Jika ada multiple extensions dengan nama method yang sama, Dart akan melaporkan error ambigu.

// File: string_extensions.dart
extension StringCaseExtensions on String {
    String capitalize() => /* ... */;
}

// File: text_utils.dart
extension TextUtilsExtensions on String {
    String capitalize() => /* ... berbeda ... */;
}

// Solusi: Import dengan prefix atau show/hide
import 'string_extensions.dart' as se;
import 'text_utils.dart' hide capitalize;

void main() {
    var text = 'hello';
    // text.capitalize(); // ERROR: ambiguous
    se.capitalize(text);  // Gunakan dengan prefix
}

💡 Gunakan prefix atau hide/show untuk mengatasi konflik.

✅ Praktik 2 – Validator Extensions

Membuat extension untuk validasi data pada String dan int.

extension StringValidator on String {
    bool get isValidEmail => RegExp(r'^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$').hasMatch(this);
    bool get isValidPhone => RegExp(r'^\+?[0-9]{10,15}$').hasMatch(this);
    bool get isNumeric => double.tryParse(this) != null;
    bool get isAlphanumeric => RegExp(r'^[a-zA-Z0-9]+$').hasMatch(this);
}

extension IntValidator on int {
    bool get isPositive => this > 0;
    bool get isNegative => this < 0;
    bool get isZero => this == 0;
    bool isBetween(int min, int max) => this >= min && this <= max;
}

void main() {
    var email = 'user@example.com';
    print(email.isValidEmail); // true

    var phone = '+628123456789';
    print(phone.isValidPhone); // true

    var numStr = '123';
    print(numStr.isNumeric); // true

    var age = 25;
    print(age.isPositive); // true
    print(age.isBetween(18, 65)); // true
}

✅ Validator extensions membuat kode validasi menjadi lebih bersih, readable, dan reusable.

📝 Ringkasan & Best Practices

Ringkasan

  • Extension methods menambahkan method ke tipe yang sudah ada tanpa modifikasi kode asli.
  • Sintaks: extension Name on Type { ... }
  • Support: method, getter, setter, operator.
  • Generic extensions pada List<T> dan tipe generic lainnya.

Best Practices

  • Nama jelas: StringExtensions, DateTimeUtils.
  • Group by type: Satu file per tipe utama.
  • Avoid overuse: Jangan jadikan semua utility sebagai extension.
  • Null safety: Handle nullable types dengan baik.
  • Dokumentasi: Tambahkan komentar untuk public extensions.

⚡ Ingat: Extension methods adalah syntactic sugar – mereka tidak mengubah tipe asli secara runtime.

🧩 Soal 1: Konflik Nama Antar Extension

SOAL Perhatikan kode berikut:

extension StringNumber on String {
  int toInt() {
    return int.parse(this);
  }
}

extension StringDouble on String {
  double toInt() {  // ← perhatikan nama method sama, return type beda
    return double.parse(this);
  }
}

void main() {
  String angka = '123';
  print(angka.toInt());
}

Pertanyaan:
Apa yang terjadi saat kode dijalankan? Jelaskan penyebabnya dan berikan solusi agar kedua extension dapat digunakan tanpa konflik.

✅ Jawaban:

Apa yang terjadi? Kode gagal dikompilasi dengan error ambiguous karena Dart tidak dapat menentukan extension mana yang dipanggil untuk method toInt() pada String. Kedua extension memiliki nama method yang sama dengan return type berbeda, tetapi Dart tidak membedakan berdasarkan return type.

Penyebab: Dart melarang pemanggilan method yang ambigu. Saat kita memanggil angka.toInt(), compiler melihat ada dua extension yang menyediakan method dengan nama yang sama, sehingga terjadi konflik.

Solusi:

  • Gunakan import dengan prefix – misalnya import 'file.dart' as ext; lalu panggil ext.toInt(angka) atau gunakan angka.toInt() dengan prefix? Sebenarnya prefix hanya berlaku untuk extension name, bukan method. Cara yang tepat adalah dengan memberi nama method yang berbeda pada salah satu extension, misalnya toIntNumber() dan toIntDouble().
  • Atau gunakan import hide/show untuk menyembunyikan salah satu extension.
  • Atau panggil secara eksplisit dengan extensionName(object).method() – misalnya StringNumber(angka).toInt() dan StringDouble(angka).toInt().

Contoh solusi dengan pemanggilan eksplisit:

void main() {
  String angka = '123';
  print(StringNumber(angka).toInt()); // memanggil dari StringNumber
  print(StringDouble(angka).toInt()); // memanggil dari StringDouble
}

🔒 Soal 2: Extension Tidak Bisa Mengakses Member Private

SOAL Diberikan class berikut:

class Counter {
  int _value = 0;  // private

  void increment() {
    _value++;
  }
}

extension CounterStats on Counter {
  int getValue() {
    return _value;  // ← baris ini bermasalah
  }
}

void main() {
  var c = Counter();
  c.increment();
  print(c.getValue());
}

Pertanyaan:
Jelaskan mengapa kode di atas gagal dikompilasi. Bagaimana cara memperbaikinya tanpa mengubah akses modifier _value menjadi publik?

✅ Jawaban:

Penyebab: Extension methods tidak dapat mengakses member private (_value) dari kelas yang diperluas. Private member hanya dapat diakses di dalam library yang sama (satu file). Karena extension berada di luar library kelas Counter, maka _value tidak terlihat.

Solusi tanpa mengubah akses modifier:

  • Buat getter publik di dalam kelas Counter yang mengembalikan nilai _value. Kemudian extension bisa menggunakan getter tersebut.
  • Atau pindahkan extension ke dalam library yang sama dengan kelas Counter (misalnya dalam satu file).

Contoh perbaikan dengan menambahkan getter publik:

class Counter {
  int _value = 0;

  int get value => _value; // tambahkan getter

  void increment() {
    _value++;
  }
}

extension CounterStats on Counter {
  int getValue() {
    return value; // sekarang bisa diakses
  }
}

void main() {
  var c = Counter();
  c.increment();
  print(c.getValue()); // output: 1
}

Dengan getter, kita tetap menjaga _value sebagai private, tetapi memberikan akses baca melalui getter yang dapat digunakan oleh extension.
